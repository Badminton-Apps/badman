# Implementation Plan: Export Foundation — Security Hardening & Shared Export Utility

**Branch**: `008-export-reports-migration` | **Date**: 2026-05-05 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/008-export-reports-migration/spec.md`

## Summary

Harden the existing unguarded `GET /excel/enrollment` REST endpoint by adding
authentication (401) and authorization (403) checks, add input validation
(400/404), and extract a shared export utility (`@badman/backend-utils`) that
provides both XLSX and CSV generation functions for reuse across all upcoming
export services (teams, exceptions, locations, avg-level).

The CP export backend (`CpController`) is already fully implemented — only
frontend integration remains, which is out of scope here.

## Phase 0: Research Findings

See [research.md](research.md) for full decision log. Key decisions:

1. **Auth pattern**: Two-step inline check in controller — `UnauthorizedException`
   (401) for no auth, `ForbiddenException` (403) for no permission. Does NOT use
   `canExecute()` (which always returns 401) — follows the `AppController.getQueueJob`
   pattern instead.

2. **Utility placement**: Added to existing `@badman/backend-utils`
   (`libs/backend/utils/src/export.util.ts`) — no new Nx project.

3. **Both formats now**: Utility exposes XLSX and CSV functions from the start
   so future specs (avg-level CSV, possible CSV alternatives) can import without
   a refactor.

4. **Validation**: Manual UUID regex + `findByPk` null check in the controller.
   Consistent with existing REST controller pattern.

5. **CP backend**: Already complete (`CpController` — `POST /cp/generate`,
   `POST /cp/webhook`, `GET /cp/download/:runId`). No backend CP work here.

## Phase 1: Design & Contracts

### Shared utility API (`export.util.ts`)

```typescript
type Row = (string | number | undefined | null)[]

// XLSX
createWorkbook(): XLSX.WorkBook
addSheet(wb, sheetName, headers, rows): void   // autofilter + auto-size included
toXlsxBuffer(wb): Buffer

// CSV — RFC 4180
toCSV(headers, rows): string      // CRLF, comma-sep, double-quote escaping
toCSVBuffer(headers, rows): Buffer
```

All functions pure — no I/O, no DI injection.

### Controller changes (`excel.controller.ts`)

```
@Get()
async getBaseplayersEnrollment(@User() user, @Res() res, @Query() query)
  1. if (!user?.id)           → throw UnauthorizedException (401)
  2. if (!hasPermission)      → throw ForbiddenException   (403)
  3. if (!isUUID(eventId))    → throw BadRequestException  (400)
  4. findByPk → if null       → throw NotFoundException    (404)
  5. call ExcelService.GetEnrollment()
  6. stream buffer with correct headers
```

### Service changes (`excel.services.ts`)

Replace the inline XLSX block with calls to the shared utility:
```
createWorkbook() → addSheet(wb, 'Enrollment', headers, rows) → toXlsxBuffer(wb)
```
Data assembly logic (player loops, sortPlayers, column calculations) is
**unchanged** — only the workbook/sheet/buffer construction is replaced.

See [contracts/enrollment-export.md](contracts/enrollment-export.md) for the
full endpoint and utility contracts.
