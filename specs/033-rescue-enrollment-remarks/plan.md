# Implementation Plan: Rescue Enrollment Remarks

**Branch**: `033-rescue-enrollment-remarks` | **Date**: 2026-06-09 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/033-rescue-enrollment-remarks/spec.md`

## Summary

Persist club admin enrollment remarks that were stranded in browser localStorage due to a frontend bug. Adds a `saveEnrollmentRemarks` GraphQL mutation that stores remarks in a new append-only `enrollment_remarks` table (event schema) and fires a notification email to `jeroen@badmintonvlaanderen.be` and `arno@dashdot.be`.

## Technical Context

**Language/Version**: TypeScript 5.x / Node 20  
**Primary Dependencies**: NestJS, Sequelize, Apollo (code-first), `@badman/backend-mailing`, `@badman/backend-notifications`  
**Storage**: PostgreSQL — new table `event.enrollment_remarks`  
**Testing**: Jest via `npx jest --config libs/backend/graphql/jest.config.ts`  
**Target Platform**: NestJS API (`apps/api`, port 5010)  
**Project Type**: web-service (GraphQL API extension)  
**Performance Goals**: Single-table insert; no throughput target (rescue tool)  
**Constraints**: Authenticated session required; no club-ownership check  
**Scale/Scope**: One-time rescue; expected < 200 club submissions
