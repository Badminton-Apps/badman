## Phase 0: Research

No unknowns. All decisions are clear from existing code patterns:

### Decision: ForbiddenException for permission denial

- **Decision**: Use `ForbiddenException` (HTTP 403) when an authenticated user lacks `export-cp:competition`.
- **Rationale**: `UnauthorizedException` (401) means "not authenticated — please provide credentials". `ForbiddenException` (403) means "authenticated but not permitted". All other export endpoints (`getTeams`, `getExceptions`, `getLocations`, `getEnrollment`) already follow this pattern.
- **Alternatives considered**: Keep `UnauthorizedException` — rejected because it sends the wrong signal to frontend clients.

### Decision: IsUUID validation on eventId

- **Decision**: Use `IsUUID(eventId)` from `@badman/utils` before any service call, throw `BadRequestException` if invalid or missing.
- **Rationale**: Identical to the pattern used in all four other export endpoints. Prevents garbage data from reaching `CpDataCollector` or the GitHub workflow dispatch.
- **Alternatives considered**: Rely on downstream validation — rejected because errors surface too late and produce confusing 502s.

### Decision: MailingService mock in test module

- **Decision**: Add `{ provide: MailingService, useValue: { sendCpExportReadyMail: jest.fn(), sendCpExportFailedMail: jest.fn() } }` to the `providers` array in `beforeEach`.
- **Rationale**: The controller's constructor injects `MailingService`; without it the NestJS DI container throws during module compilation, failing all 17 tests before any assertion runs.
- **Alternatives considered**: Remove `MailingService` from the controller — rejected because email notification is a required feature.

## Phase 1: Design

### Data Model

No changes. The in-memory `Map<string, CpGenerationRecord>` is intentional for a once-a-year operation and documented as such in the controller.

### API Contracts

The three endpoints remain unchanged in signature. The only observable change is:

| Endpoint | Path | Before | After |
|----------|------|--------|-------|
| `POST /cp/generate` | Permission denied | 401 | 403 |
| `POST /cp/generate` | Invalid/missing `eventId` | 400 (raw `HttpException`) | 400 (`BadRequestException`) |
| `GET /cp/download/:runId` | Permission denied | 401 | 403 |

Response body shape and all other status codes are unchanged.

### Files to Modify

#### `apps/api/src/app/controllers/cp.controller.ts`

1. Add `ForbiddenException` to NestJS imports.
2. In `generate()`: change `throw new UnauthorizedException(...)` for permission check to `throw new ForbiddenException('Insufficient permissions')`.
3. In `generate()`: after auth checks, add `if (!eventId || !IsUUID(eventId)) throw new BadRequestException('eventId must be a valid UUID')`.
4. Replace `throw new HttpException('eventId is required', 400)` with `throw new BadRequestException('eventId is required')`.
5. Add `import { IsUUID } from '@badman/utils'`.
6. In `download()`: change `throw new UnauthorizedException(...)` for permission check to `throw new ForbiddenException('Insufficient permissions')`.

#### `apps/api/src/app/controllers/cp.controller.spec.ts`

1. Add `MailingService` import from `@badman/backend-mailing`.
2. In `beforeEach`, add to `providers`: `{ provide: MailingService, useValue: { sendCpExportReadyMail: jest.fn(), sendCpExportFailedMail: jest.fn() } }`.
3. Update the two `rejects.toThrow(UnauthorizedException)` assertions for permission denial to `rejects.toThrow(ForbiddenException)` (import `ForbiddenException` alongside `UnauthorizedException`).
